Listen und Datenstrukturen – Teil 1

Array, Linked List, Stack und Queue – die vier wichtigsten Bausteine für die Organisation von Daten

WS 2025/26  ·  DHBW Stuttgart

01

Lernziele

📌 Nach dieser Vorlesung kannst du …

  • erklären, warum Datenstrukturen existieren und welchen Vorteil sie gegenüber einzelnen Variablen bieten.
  • den Unterschied zwischen Array, Linked List, Stack und Queue beschreiben.
  • die Prinzipien LIFO und FIFO erklären und für eine gegebene Situation die passende Datenstruktur wählen.
02

Warum brauchen wir Datenstrukturen?

Stell dir vor, du schreibst ein Programm, das die Messwerte von 100 Temperatursensoren verwaltet. Ohne Datenstruktur bräuchtest du 100 einzelne Variablen: sensor1, sensor2, … sensor100. Das ist nicht nur mühsam zu schreiben – es lässt sich auch nicht automatisch verarbeiten. Kommt ein 101. Sensor dazu, muss der gesamte Code manuell erweitert werden.

Eine Datenstruktur löst dieses Problem: Sie fasst mehrere Datenwerte in einem einzigen, strukturierten Behälter zusammen. Die Auswahl der richtigen Datenstruktur entscheidet darüber, ob Operationen wie Einfügen, Suchen oder Löschen schnell und einfach oder langsam und fehleranfällig sind.

Ohne Datenstruktur

100 einzelne Variablen. Für jede Berechnung muss jede Variable einzeln angefasst werden. Eine Schleife ist nicht möglich. Wartung ist ein Albtraum.

Mit Datenstruktur

Alle 100 Werte in einer einzigen Struktur. Eine Schleife verarbeitet sie automatisch. 1000 Werte funktionieren mit demselben Code. Erweiterung ist trivial.

Praxisbezug Maschinenbau

In der Maschinenüberwachung (Condition Monitoring) werden kontinuierlich Sensorwerte gesammelt und ausgewertet. Eine Warteschlange (Queue) speichert die zuletzt gemessenen Werte, ein Array hält die Referenzwerte vor – ohne die richtigen Datenstrukturen wäre moderne Maschinendiagnose nicht realisierbar.

03

Array / Liste – Nummerierte Ablagefächer

Ein Array (oder Liste) ist die grundlegendste Datenstruktur. Sie speichert eine Folge von Elementen an direkt aufeinanderfolgenden Speicherplätzen. Jedes Element besitzt einen Index – eine Nummer, über die es direkt angesprochen werden kann. Der Index beginnt in den meisten Systemen bei 0.

Visualisierung: Ein Array mit 5 Elementen

Das rot markierte Element an Index 0 hat den Wert 42 und ist über seinen Index direkt erreichbar – unabhängig davon, wie groß das Array ist.

Vorteile des Arrays

  • Direkter Zugriff über Index – sehr schnell, egal ob Element 1 oder Element 1.000.000.
  • Einfache Iteration: Alle Elemente können mit einer Schleife durchlaufen werden.
  • Speichereffizient: Elemente liegen direkt nebeneinander im Arbeitsspeicher.

Nachteile des Arrays

  • Einfügen in der Mitte ist aufwendig: Alle nachfolgenden Elemente müssen verschoben werden.
  • Die Größe kann je nach Systemimplementierung fest vorgegeben sein.
  • Löschen erzeugt „Lücken", die verwaltet werden müssen.
04

Linked List – Die verkettete Liste

Eine Linked List (verkettete Liste) speichert Elemente nicht an aufeinanderfolgenden Speicherplätzen, sondern als unabhängige „Knoten". Jeder Knoten enthält seinen Datenwert und einen Verweis (Zeiger) auf den nächsten Knoten. Die Elemente sind durch diese Verweise miteinander verkettet – wie Waggons eines Zuges, die durch Kupplungen verbunden sind.

Analogie: Der Zug

Stell dir einen Zug vor: Jeder Waggon ist eigenständig und enthält Passagiere (Daten). Über die Kupplung weiß er, welcher Waggon als nächstes kommt. Du kannst einen neuen Waggon einfach in der Mitte einkoppeln, ohne den Rest des Zuges zu verschieben. Beim Array müsstest du dagegen alle dahinterliegenden Waggons physisch nach hinten rücken.

Jeder Knoten besteht aus einem Datenwert (linkes Feld) und einem Zeiger auf den nächsten Knoten (rechtes Feld). Der letzte Knoten zeigt auf ∅ (Null / kein Nachfolger).

Vorteile der Linked List

  • Einfügen und Löschen in der Mitte ist schnell: Nur die Zeiger müssen angepasst werden, keine Elemente müssen verschoben werden.
  • Flexible Größe: Die Liste kann beliebig wachsen, solange Arbeitsspeicher verfügbar ist.

Nachteile der Linked List

  • Kein direkter Zugriff über Index: Um das 50. Element zu finden, muss man von vorne durch die ganze Kette laufen.
  • Jeder Knoten benötigt zusätzlichen Speicher für den Zeiger.
05

Stack – Der Tellerstapel (LIFO)

Ein Stack (Stapel) ist eine Datenstruktur, bei der Elemente immer oben hinzugefügt und oben wieder entnommen werden. Das zuletzt hinzugefügte Element wird als erstes wieder herausgenommen. Dieses Prinzip heißt LIFO – Last In, First Out (Zuletzt rein, zuerst raus).

Die beiden einzigen erlaubten Operationen auf einem Stack lauten: push (Element oben auflegen) und pop (oberstes Element entnehmen).

Visualisierung: Tellerstapel

Stack in der Praxis

Der Stack ist überall dort die richtige Wahl, wo die Reihenfolge umgekehrt werden soll oder wo man die letzte Aktion rückgängig machen muss.

Praxisbeispiel: Undo im Texteditor

Wenn du in einem Texteditor „Rückgängig" drückst, wird die zuletzt ausgeführte Aktion als erstes rückgängig gemacht – genau das LIFO-Prinzip. Jede Aktion wird auf den Stack gelegt (push). Rückgängig bedeutet: oberstes Element entnehmen (pop).

Weiteres Beispiel: Browser-Zurück-Taste

Beim Surfen wird jede besuchte Seite auf einen Stack gelegt. Drückst du „Zurück", wird die zuletzt besuchte Seite (oberstes Stack-Element) angezeigt – klassisches LIFO.

06

Queue – Die Warteschlange (FIFO)

Eine Queue (Warteschlange) ist eine Datenstruktur, bei der Elemente hinten hinzugefügt und vorne entnommen werden. Das zuerst hinzugefügte Element wird auch als erstes wieder herausgenommen. Dieses Prinzip heißt FIFO – First In, First Out (Zuerst rein, zuerst raus).

Die zwei Operationen heißen enqueue (Hinten einreihen) und dequeue (Vorne entnehmen). Das entspricht der natürlichen Warteschlange: Wer zuerst ansteht, wird auch zuerst bedient.

Visualisierung: Warteschlange am Bäcker

Praxisbeispiel: Druckerwarteschlange

In einem Büro werden Druckaufträge nacheinander in eine Queue gelegt. Der erste Auftrag, der ankam, wird auch als erster gedruckt – unabhängig davon, wer ihn abgeschickt hat. Das ist FIFO im Einsatz: fair und geordnet.

Weiteres Beispiel: Datenübertragung

In Netzwerken werden Datenpakete in Queues gepuffert. Das erste ankommende Paket wird als erstes weitergeleitet. Ohne Queue würden Pakete durcheinander kommen und die Daten korrumpiert.

07

Welche Datenstruktur für welche Situation?

Die Wahl der richtigen Datenstruktur ist eine der wichtigsten Designentscheidungen in der Softwareentwicklung. Die folgende Entscheidungshilfe fasst die vier Strukturen aus dieser Vorlesung zusammen.

DatenstrukturPrinzipStärkeWenn du …
Array / Liste Direkter Index-Zugriff Schnelles Lesen über Index … viele Werte speichern und häufig über Index zugreifen willst.
Linked List Verkettete Zeiger Schnelles Einfügen / Löschen in der Mitte … häufig Elemente mitten in der Liste einfügst oder entfernst.
Stack LIFO (Letzter rein, erster raus) Letzte Aktion rückgängig machen … Aktionen aufzeichnest und in umgekehrter Reihenfolge verarbeitest (Undo, Browser-History).
Queue FIFO (Erster rein, erster raus) Aufgaben in Ankunftsreihenfolge abarbeiten … Aufgaben fair in der Reihenfolge ihres Eintreffens verarbeitest (Druckerwarteschlange, Netzwerkpuffer).
08

Zusammenfassung

Merksatz

Ein Array speichert Elemente nebeneinander und erlaubt direkten Zugriff über den Index. Eine Linked List verkettet Knoten über Zeiger und ermöglicht flexibles Einfügen und Löschen. Der Stack arbeitet nach dem LIFO-Prinzip (letztes Element zuerst raus) – push und pop. Die Queue arbeitet nach dem FIFO-Prinzip (erstes Element zuerst raus) – enqueue und dequeue. Die richtige Wahl der Datenstruktur ist eine Frage des Verwendungszwecks.
