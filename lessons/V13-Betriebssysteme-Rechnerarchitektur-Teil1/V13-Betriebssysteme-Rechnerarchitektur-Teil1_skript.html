02

Von-Neumann-Architektur

Fast alle Computer, die heute gebaut werden, basieren auf einem Konzept, das John von Neumann 1945 beschrieben hat: dem sogenannten Stored-Program-Konzept. Die entscheidende Idee: Programm und Daten liegen im selben Speicher – die CPU liest beides zur Laufzeit und führt es aus. Vor diesem Konzept mussten Programme per Hand in Hardware verdrahtet werden (z. B. ENIAC, 1945).

1 – Eingabe (Input)

Tastatur, Maus, Kamera, Sensoren – alles, womit Daten in den Rechner gelangen. Ohne Eingabe gibt es keine Verarbeitung.

2 – Ausgabe (Output)

Bildschirm, Lautsprecher, Drucker, Aktoren – alles, womit Ergebnisse nach außen gegeben werden.

3 – Speicher (Memory)

Hier liegen sowohl die zu verarbeitenden Daten als auch das laufende Programm. Das ist der Kern des Stored-Program-Konzepts: Programm und Daten im gleichen Speicher, adressierbar per Adresse.

4 – Rechenwerk (ALU)

Die Arithmetic Logic Unit führt alle arithmetischen Operationen (Addition, Subtraktion) und logischen Verknüpfungen (AND, OR, NOT) aus. Sie ist das eigentliche „Rechnen" im Rechner.

5 – Steuerwerk (Control Unit)

Das Steuerwerk koordiniert alle anderen Komponenten. Es liest Befehle aus dem Speicher, dekodiert sie und steuert die ALU, den Speicher und die Ein-/Ausgabe entsprechend.

Merksatz

CPU = Steuerwerk + Rechenwerk. Zusammen mit Speicher, Eingabe und Ausgabe bilden sie die fünf Komponenten der Von-Neumann-Architektur – die Grundlage aller modernen Computer.

03

Fetch – Decode – Execute: Was die CPU tut

Eine CPU führt nicht einfach ein gesamtes Programm auf einmal aus. Sie arbeitet in einem endlosen, extrem schnellen Zyklus, der immer dieselben drei Schritte wiederholt: Befehl holen, Befehl entschlüsseln, Befehl ausführen.

Schritt 1: Fetch – Holen

Das Steuerwerk liest den nächsten Befehl aus dem Arbeitsspeicher. Die Adresse des aktuellen Befehls wird im Programmzähler gespeichert, der nach jedem Fetch automatisch weiterrückt.

Schritt 2: Decode – Entschlüsseln

Das Steuerwerk interpretiert den geholten Befehl: Was soll getan werden? Addieren? Speichern? Springen? Es übersetzt den Maschinencode in konkrete Steuersignale für die nachfolgenden Einheiten.

Schritt 3: Execute – Ausführen

Die ALU oder eine andere Funktionseinheit führt den Befehl aus. Das Ergebnis wird in ein Register oder zurück in den Arbeitsspeicher geschrieben. Dann beginnt der Zyklus von vorne.

Größenordnung

Ein moderner Prozessor führt diesen Zyklus mehrere Milliarden Mal pro Sekunde aus. Ein 3-GHz-Prozessor macht theoretisch 3.000.000.000 Takte pro Sekunde. Pro Takt können sogar mehrere Befehle parallel abgearbeitet werden – das nennt sich Pipelining.

04

Speicherhierarchie – Warum nicht alles schnell sein kann

Schneller Speicher ist teuer und braucht viel Energie. Großer Speicher ist günstig, aber langsam. Deshalb löst kein Computerhersteller das Problem mit einem einzigen Speichertyp – stattdessen gibt es eine Speicherhierarchie: Kleine, extrem schnelle Speicher nah an der CPU; große, langsame und günstige Speicher weiter weg.

Praxisbeispiel: Warum friert mein PC ein?

Wenn du 20 Browser-Tabs + Zoom + ein Spiel gleichzeitig offen hast, füllt sich der RAM. Das Betriebssystem beginnt dann, Inhalte auf die viel langsamere SSD auszulagern (sogenanntes Swapping). Der Zugriff auf die SSD ist hunderte Male langsamer als RAM. Das Ergebnis ist spürbare Verzögerung – der Rechner „friert" aus Nutzersicht ein.

05

Cache-Hierarchie: L1, L2 und L3

Der Cache ist ein schneller Zwischenspeicher direkt auf dem Prozessorchip. Da die CPU viel schneller arbeitet als der RAM, würde sie ständig warten, wenn sie jeden Datenwert direkt aus dem RAM holen müsste. Der Cache löst dieses Problem, indem er häufig benötigte Daten in unmittelbarer Nähe der Rechenkerne bereithält.

EbeneGröße (typisch)ZugriffszeitBesonderheit
L1-Cache 32–64 KB pro Kern ~1 ns Direkt im CPU-Kern; getrennt für Daten und Befehle; extrem schnell
L2-Cache 256 KB – 1 MB pro Kern ~5 ns Im CPU-Chip, aber nicht mehr direkt im Kern; gemeinsame Pufferzone
L3-Cache 8–32 MB, alle Kerne geteilt ~20–40 ns Größter On-Chip-Cache; von allen Kernen gemeinsam genutzt
RAM 8–64 GB ~60–100 ns Außerhalb des Chips; deutlich langsamer, aber viel größer
Cache-Hit vs. Cache-Miss

Wenn die CPU einen Datenwert braucht und dieser bereits im Cache liegt, spricht man von einem Cache-Hit – der Zugriff ist blitzschnell. Liegt er nicht im Cache (Cache-Miss), muss die CPU aus dem langsameren RAM oder noch weiter unten in der Hierarchie holen. Gut strukturierter Code erhöht die Cache-Hit-Rate erheblich.

06

Mehrkernprozessoren und GPUs

Die Taktfrequenz einer CPU lässt sich physikalisch nicht beliebig steigern – ab einer gewissen Frequenz erzeugen die Transistoren so viel Wärme, dass der Chip beschädigt wird. Die Lösung der Hersteller seit etwa 2005: statt einer schnelleren CPU mehrere Kerne auf demselben Chip. Ein 8-Kern-Prozessor kann in der Theorie acht Aufgaben gleichzeitig bearbeiten.

CPU – wenige starke Kerne

Eine moderne CPU hat 4 bis 32 Kerne, die jeweils sehr leistungsstark sind. Sie eignet sich für komplexe, sequenzielle Aufgaben – wie das Ausführen eines Betriebssystems oder komplizierter Berechnungen mit vielen Abhängigkeiten.

GPU – tausende einfache Kerne

Eine GPU (Graphics Processing Unit) enthält tausende einfache, parallele Kerne. Sie wurde ursprünglich für Grafik entwickelt, wo dieselbe Berechnung auf millionen von Pixeln gleichzeitig angewendet werden muss. Heute wird dieselbe Parallelisierung für KI-Training und Simulationen genutzt.

Analogie

Eine CPU ist wie ein Team aus 8 hochqualifizierten Spezialisten – jeder kann komplexe Aufgaben selbstständig lösen. Eine GPU ist wie eine Fabrikhalle mit 10.000 einfachen Arbeitern, die alle dieselbe Bewegung gleichzeitig ausführen. Für Fließbandarbeit (Pixel berechnen, Matrix-Multiplikationen für KI) gewinnt die Fabrikhalle haushoch.

07

Zusammenfassung: Rechnerarchitektur

Merksatz

Die Von-Neumann-Architektur beschreibt den Aufbau aller modernen Computer in fünf Komponenten: Eingabe, Ausgabe, Speicher, Rechenwerk und Steuerwerk. Die CPU arbeitet im Fetch-Decode-Execute-Zyklus mit Milliarden von Takten pro Sekunde. Die Speicherhierarchie von Register bis HDD erklärt, warum Geschwindigkeit und Größe im Widerspruch stehen – und warum zu viele offene Programme den Rechner verlangsamen.
