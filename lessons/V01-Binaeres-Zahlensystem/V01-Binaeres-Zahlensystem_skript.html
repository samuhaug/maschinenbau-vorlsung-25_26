Binäres Zahlensystem

Wie Computer mit Nullen und Einsen rechnen – und warum das für Ingenieure relevant ist

02

Was ist ein Bit?

Der kleinste Informationsträger in einem Computer heißt Bit (von binary digit, deutsch: binäre Stelle). Ein Bit kann genau zwei Zustände annehmen: 0 oder 1. In der Hardware entspricht das zwei definierten Spannungspegeln – typischerweise 0 V (aus) und 3,3 V (an).

Acht Bits zusammen bilden ein Byte. Ein Byte kann 2⁸ = 256 verschiedene Werte darstellen (von 0 bis 255). Alle Dateigrößen, Arbeitsspeicher-Angaben und Speicherkapazitäten werden in Vielfachen von Byte ausgedrückt: Kilobyte (kB), Megabyte (MB), Gigabyte (GB).

Stellenwert-Darstellung: 1011₂ = 11₁₀

Jedes Bit repräsentiert einen Stellenwert – eine Zweierpotenz. Aktive Bits (Wert 1) addieren ihren Stellenwert zum Gesamtwert; inaktive Bits (Wert 0) tragen nichts bei.

Rot markierte Bits (Wert 1) aktivieren ihren Stellenwert. Summe: 8 + 0 + 2 + 1 = 11₁₀.

Merkhilfe

Stell dir jeden Bit-Schalter wie eine Lampe vor: Leuchtet sie (Wert 1), zählt ihr Stellenwert zur Gesamtsumme. Ist sie aus (Wert 0), trägt sie nichts bei.

03

Stellenwertsysteme

Jedes Stellenwertsystem basiert auf einer Basis. Im vertrauten Dezimalsystem (Basis 10) ist jede Stelle eine Zehnerpotenz wert. Das Binärsystem (Basis 2) funktioniert exakt gleich – nur mit Zweierpotenzen statt Zehnerpotenzen.

Dezimalsystem (Basis 10)

Die Zahl 345 bedeutet:

3 · 10² + 4 · 10¹ + 5 · 10⁰
= 300 + 40 + 5 = 345

Binärsystem (Basis 2)

Die Zahl 1011₂ bedeutet:

1 · 2³ + 0 · 2² + 1 · 2¹ + 1 · 2⁰
= 8 + 0 + 2 + 1 = 11₁₀

Die wichtigsten Zweierpotenzen

Dezimalwert 124816 32641282561 024
Praxishinweis

2¹⁰ = 1 024 ≈ 1 000. Deshalb ist 1 Kibibyte (KiB) = 1 024 Byte, nicht exakt 1 000. Daher erscheint eine „512 GB"-Festplatte im Betriebssystem etwas kleiner – der Hersteller zählt in 10er-Potenzen (1 GB = 1 000 000 000 Byte), das Betriebssystem in 2er-Potenzen.

04

Binär ↔ Dezimal Umrechnung

Binär → Dezimal: Stellenwerte addieren

Jede Stelle wird mit ihrer Zweierpotenz multipliziert, alle Produkte werden addiert. Es ist dieselbe Methode, mit der man im Dezimalsystem „liest" – nur mit Zweierpotenzen als Stellenwerte.

Beispiel: 10110₂ → ?₁₀

1 · 2⁴ + 0 · 2³ + 1 · 2² + 1 · 2¹ + 0 · 2⁰
= 16 + 0 + 4 + 2 + 0 = 22₁₀

Dezimal → Binär: Wiederholte Division durch 2

Die Dezimalzahl wird so oft durch 2 geteilt, bis 0 übrig bleibt. Die Reste (immer 0 oder 1) werden notiert. Der erste Rest ist das LSB (niederwertigstes Bit, rechts), der letzte Rest ist das MSB (höchstwertigstes Bit, links). Die Reste von unten nach oben gelesen ergeben die Binärdarstellung.

Beispiel: 13₁₀ → ?₂
DivisionGanzzahl-ErgebnisRest
13 ÷ 261 ← LSB
6 ÷ 230
3 ÷ 211
1 ÷ 201 ← MSB

Reste von unten nach oben lesen: 1101₂. Probe: 8 + 4 + 0 + 1 = 13 ✓

Begriffe: MSB und LSB

MSB (Most Significant Bit) ist das höchstwertige Bit – ganz links, analog zur Tausender-Stelle im Dezimalsystem. LSB (Least Significant Bit) ist das niederwertigste Bit – ganz rechts, analog zur Einer-Stelle. Diese Begriffe begegnen dir später oft in Datenblättern von Mikrokontrollern und Sensoren.

05

Hexadezimalsystem

Das Hexadezimalsystem (Basis 16) verwendet die Ziffern 0–9 sowie die Buchstaben A–F für die Werte 10–15. Der entscheidende Vorteil: Genau 4 Bits lassen sich als eine einzige Hex-Ziffer schreiben. Ein Byte (8 Bits) entspricht damit immer exakt zwei Hex-Ziffern. Das macht Hex zur bevorzugten Schreibweise für Speicheradressen, Farben und Protokoll-Daten.

Dezimal 012…9 101112131415
Hex 012…9 ABCDEF
Binär 000000010010…1001 101010111100110111101111
Umrechnung: A3₁₆ → Dezimal

A · 16¹ + 3 · 16⁰ = 10 · 16 + 3 · 1 = 160 + 3 = 163₁₀

Praxisbeispiel – Farbcodes am Smartphone

Jede Bildschirmfarbe setzt sich aus drei Farbkanälen zusammen: Rot, Grün und Blau (RGB). Jeder Kanal wird durch einen Wert von 0 bis 255 beschrieben – das ist exakt ein Byte. Im Hex-Format werden alle drei Bytes direkt hintereinander geschrieben, mit einem # davor.

#FF5733 – Rot = FF₁₆ = 255, Grün = 57₁₆ = 87, Blau = 33₁₆ = 51 → kräftiges Orange-Rot.

#1DA1F2 – Rot = 1D₁₆ = 29, Grün = A1₁₆ = 161, Blau = F2₁₆ = 242 → das bekannte Twitter/X-Blau.

Da jeder Kanal 256 Abstufungen hat und drei Kanäle kombiniert werden, lassen sich 256³ = 16 777 216 Farben darstellen – diesen Standard nennt man True Color. Jetzt weißt du, was hinter jedem Farbcode in Figma, CSS oder Instagram steckt.

06

Binäre Addition

Die binäre Addition folgt demselben Prinzip wie die schriftliche Dezimaladdition. Es gibt nur vier mögliche Fälle – je zwei Eingangs-Bits ergeben eine Summe und einen Übertrag (englisch: carry):

Bit ABit BSummeÜbertrag (Carry)
0000
0110
1010
1101

Der Übertrag entsteht, weil 1 + 1 = 2 wäre – aber die Ziffer 2 existiert im Binärsystem nicht. Stattdessen schreibt man an der aktuellen Stelle eine 0 und überträgt eine 1 an die nächsthöhere Stelle. Das ist identisch mit dem Zehnerübertrag beim Dezimaladdieren.

Beispiel: 1011₂ + 0110₂

1011₂ + 0110₂ = 10001₂. Probe: 11 + 6 = 17₁₀ = 16 + 1 = 1·2⁴ + 1·2⁰ = 10001₂ ✓

Überlauf (Overflow)

Wenn das Ergebnis mehr Bits benötigt als das Register breit ist, spricht man von einem Überlauf. Ein 4-Bit-Addierwerk kann Werte nur bis 15 darstellen. Bei 11 + 6 = 17 wird das 5. Bit abgeschnitten; im Register steht dann fälschlicherweise 1. Dieses Problem ist in der Mikrocontroller-Programmierung (z. B. Arduino, STM32) ein klassischer Fehler, den du kennen musst.

07

Negative Zahlen – das Zweierkomplement

Computer müssen auch negative Zahlen speichern. Der naheliegendste Ansatz wäre ein Vorzeichenbit: das erste Bit zeigt 0 für positiv, 1 für negativ. Dieser Ansatz hat jedoch zwei Probleme:

  • Die Zahl Null wäre doppelt vorhanden: +0 und −0 wären unterschiedliche Bitmuster, obwohl sie denselben Wert darstellen.
  • Die CPU müsste intern prüfen, ob ein Vorzeichen-Bit gesetzt ist, und je nachdem anders rechnen – das erfordert extra Schaltlogik und macht die Hardware komplizierter.

Die Lösung, die heute in allen modernen Prozessoren verwendet wird, heißt Zweierkomplement. Die Kernidee ist, negative Zahlen so zu kodieren, dass die normale Additions-Schaltung der CPU für positive und negative Zahlen gleich funktioniert – ohne jede Sonderbehandlung.

Intuition: die Uhren-Analogie

Stell dir eine Uhr vor: Zeiger auf 12 entspricht dem Wert 0. Um „−1" darzustellen, drehst du um eine Position zurück – der Zeiger steht auf 11. In einem 4-Bit-Register entspricht das dem Muster 1111. Wenn man jetzt 1111 + 0001 addiert, entsteht 10000 – das 5. Bit passt nicht in das 4-Bit-Register und fällt weg. Es bleibt 0000. Die Addition von −1 und +1 ergibt also korrekt 0.

Warum ist das für Maschinenbau-Ingenieure relevant?

In Sensor-Datenblättern und Kommunikationsprotokollen begegnen dir vorzeichenbehaftete Datentypen (engl. signed) und vorzeichenlose Typen (engl. unsigned). Wer nicht versteht, dass negative Zahlen kein einfaches Vorzeichen-Bit sind, begeht klassische Bugs beim Auslesen von Temperatursensoren, Winkelsensoren (IMU) oder CAN-Bus-Nachrichten – zum Beispiel liest man −10 °C als 65 526 aus.

08

Zusammenfassung

Merksätze V01
  • Ein Bit ist die kleinste Informationseinheit mit den Werten 0 oder 1. Acht Bits bilden ein Byte mit 256 möglichen Werten.
  • Im Binärsystem (Basis 2) hat jede Stelle doppelt so viel Gewicht wie die rechte Nachbarstelle – statt der Zehnerpotenz gilt die Zweierpotenz.
  • Das Hexadezimalsystem (Basis 16) komprimiert 4 Bits auf eine einzige Stelle; ein Byte entspricht immer genau zwei Hex-Ziffern.
  • Binäre Addition funktioniert wie dezimale Addition: Bei 1 + 1 entsteht ein Übertrag an die nächste Stelle. Überschreitet das Ergebnis die Bitbreite, tritt ein Überlauf auf.
  • Computer speichern negative Zahlen im Zweierkomplement, damit dieselbe Additionsschaltung für positive und negative Zahlen gleichermaßen funktioniert – ohne Sonderlogik.
