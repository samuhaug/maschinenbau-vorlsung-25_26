Operatoren im Überblick

Vergleichsoperatoren – Ergebnis ist immer True oder False

==ist gleich
!=ungleich

>größer als
<=kleiner oder gleich
>=größer oder gleich

Logische Operatoren – Bedingungen verknüpfen

andbeide wahr → wahr
ormind. eine wahr → wahr
notWahrheitswert umkehren
= vs. == – der häufigste Anfängerfehler

= ist eine Zuweisung: temperatur = 23.5 – speichert den Wert.
== ist ein Vergleich: temperatur == 23.5 – prüft, ob der Wert gleich ist.

03

Operatoren im Detail

Vergleichsoperatoren

Jeder Vergleich ergibt True oder False. Das Ergebnis kann in einer Variable gespeichert oder direkt in einer Bedingung verwendet werden.

temperatur = 85.3 grenzwert = 80.0 print(temperatur > grenzwert) # → True print(temperatur == 80.0) # → False print(temperatur != 80.0) # → True print(temperatur <= 100.0) # → True # Ergebnis in Variable speichern alarm = temperatur > grenzwert # True print("Alarm:", alarm) # Wertebereich prüfen (Python-Besonderheit) in_bereich = 0 <= temperatur <= 100 # True
and · or · not

Mit and und or verknüpfst du mehrere Bedingungen. not kehrt einen Wahrheitswert um.

temp = 85.3 druck = 1.05 notaus = False # and: beide Bedingungen müssen wahr sein alarm = temp > 80 and druck > 1.0 print("Druckalarm + Temperaturalarm:", alarm) # True # or: mindestens eine Bedingung muss wahr sein irgendein_alarm = temp > 100 or druck > 1.0 print("Irgendein Alarm:", irgendein_alarm) # True # not: Wahrheitswert umkehren motor_kann_starten = not notaus print("Motor kann starten:", motor_kann_starten) # True # Komplexe Bedingung sicher = not notaus and temp < 100 and druck < 2.0
Klammern für Lesbarkeit

Bei komplexen Bedingungen helfen Klammern: (temp > 80) and (druck > 1.0) ist genauso korrekt wie ohne Klammern – aber deutlich leichter zu lesen.
